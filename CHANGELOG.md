# Changelog

All notable changes to Open Work Standard are documented here.

## [0.1.1] — 2026-07-13

### Added

- Filesystem activity-pattern review signals for concentrated bursts, bulk creation, compressed duration, low revision density, and late rewrites.
- Offline detection of explicit authorship markers such as “generated by ChatGPT” or “written by Claude”.
- Activity metrics in text and JSON verification reports and package inspection output.
- Tests covering normal incremental work, instant-generated projects, baseline filtering, late rewrites, and signed-package review signals.

### Changed

- Signed packages remain cryptographically `Verified` while clearly recommending activity or authorship review when signals are present.
- Windows Installed Apps now reports version `0.1.1`.
- README documents the privacy boundary: OWS does not perform keylogging, screen capture, clipboard monitoring, hidden process surveillance, or direct AI/LLM-use detection.

### Validation

- `dotnet format --verify-no-changes --severity warn`
- `dotnet build OWS.sln -c Release --no-restore -nologo`
- `dotnet test OWS.sln -c Release --no-build -nologo`
- `git diff --check`

### Release artifact hashes

Hashes are SHA-256 values. Reproduce them after building the Windows installer with:

```powershell
Get-FileHash .\artifacts\ows-setup\Ows.Setup.exe -Algorithm SHA256
```
